# マーケティングPDCA設計
*v1.0 / 2026-05-20*

---

## 現状の課題認識

| 課題 | 詳細 |
|---|---|
| LINE登録母数が少ない | SNS→LINEへの誘導が弱い or SNSリーチが足りない |
| 応募数が取れていない | LINE登録→応募への転換率が低い |
| 安定した成果が見えにくい | まだデータ蓄積フェーズ |

---

## ファネル設計（現状と目標）

```
SNSリーチ（認知）
    ↓ [転換率目標: 0.5%以上]
プロフィール訪問・LINE誘導タップ
    ↓ [転換率目標: 30%以上]
LINE登録
    ↓ [転換率目標: 50%以上]
応募意向・説明会参加
    ↓ [転換率目標: 20%以上]
入社
```

**現状の推定ボトルネック:** SNSリーチが少ない → まずリーチを増やすことが最優先

## PDCA サイクル設計

### Planフェーズ（月初）

**確認事項:**
- 先月のKPI実績（KPIシート参照）
- 先月うまくいった投稿の共通点
- 先月リーチが低かった投稿の共通点
- 今月の重点テーマ決定

**teaへの依頼例:**
```
「先月の投稿データを共有するから
 うまくいった投稿の共通点を分析して
 今月の改善施策を3つ提案して」
```

---

### Doフェーズ（月中）

**施策の優先順位（現フェーズ）:**

| 優先度 | 施策 | 目的 |
|---|---|---|
| ★★★ | 投稿本数を増やす（週3本以上） | SNSリーチの絶対量を増やす |
| ★★★ | 冒頭3秒のフックを毎回改善 | 視聴維持率向上 |
| ★★☆ | LINEへのCTAを毎投稿に入れる | 登録への誘導強化 |
| ★★☆ | コメント・DM返信を24時間以内に | エンゲージメント向上 |
| ★☆☆ | ハッシュタグを毎週A/Bテスト | リーチ最大化 |

---

### Checkフェーズ（毎週月曜）

**週次チェック項目:**

| 指標 | 確認内容 |
|---|---|
| リーチ数 | 先週比で増えているか |
| 保存数・シェア数 | バズった投稿はないか |
| LINE登録数 | 今週何人増えたか |
| コメント傾向 | ポジティブ/ネガティブの比率 |

**フォーマット（週報に記録）:**
```
📊 今週のチェック
・最高リーチ投稿：[URL] → [数値]リーチ → 理由：[分析]
・LINE登録：[数値]名
・来週改善すること：[1つだけ決める]
```

---

### Actionフェーズ（毎週金曜）

**改善の型（3つから選んで実行）:**

**A：コンテンツ改善**
- 「バズった投稿」の型を再現する
- 「スベった投稿」の冒頭を変えてリポスト

**B：導線改善**
- プロフィール文のCTAを変える
- LINEへの誘導方法を変える（QRコード位置・テキスト）

**C：配信タイミング改善**
- 投稿時間をターゲットの活動時間に合わせる
- TikTok：20〜23時が多くの場合ベスト

## LINE登録→応募 転換施策

### 現状のLINE運用ルール
- 登録通知が来たら24時間以内に初回メッセージ送信
- 職場体験会・説明会の案内を定期配信
- 質問には丁寧に・素早く返答

### 転換率を上げるための施策

| 施策 | 内容 | teaへの指示 |
|---|---|---|
| ステップ配信設計 | 登録後3日以内に自動で3通送るシナリオ | 「LINE登録後のステップ配信シナリオ3通作って」 |
| よくある質問集 | 求職者の不安を先回りして解消 | 「採用LINEのFAQ20問作って」 |
| 説明会の定期化 | 月2回固定で開催→参加ハードルを下げる | 「説明会の告知メッセージ作って」 |
| 体験談コンテンツ | LINEで「入社者の声」を送る | 「入社者の声テキストコンテンツ作って」 |
